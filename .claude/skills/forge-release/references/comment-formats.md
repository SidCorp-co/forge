# forge-release comment formats

All comments posted via `forge_comments → create`. Always English (see [`../../README.md` § English-only rule](../../README.md)).

## On successful merge + close (Step 7)

```markdown
## Released — ISS-XX

Merged to `<TARGET>` as `<merge-hash>`.
Branch + worktree cleaned up.

Local E2E verdict (forge-test): PASS (or soft-pass with `e2e-not-verified`)

🤖 Generated by forge-release (TBD merge + auto-close)
```

## On merge conflict (Step 3)

After `git merge --abort` + `released → reopen`:

```markdown
**Release merge conflict** — `<TARGET>` moved between forge-test and forge-release.
Conflicting files:
  - <list>

Run /forge-fix <documentId> to rebase ISS-XX-short-title onto `<TARGET>` and re-trigger forge-test.
```

## On post-merge test failure (Step 4)

After `git reset --hard HEAD~1` + `released → reopen`:

```markdown
**Release test failure** — new failures appeared after merging ISS-XX-short-title into `<TARGET>`.
Likely cause: another release landed between forge-test and forge-release.

Failing tests:
  - <one line per failing test with the top assertion>

Reset the merge. Run /forge-fix <documentId> to rebase + re-test.
```

## On forge-test fail verdict (Step 1 — pre-merge guard)

If the latest `forge-test` comment reports FAIL, do not merge:

```markdown
Cannot release — last forge-test verdict was FAIL. Run forge-fix first.
```

Leave status at `released` for human inspection. Do NOT transition or merge.

## On busy target worktree (Step 2)

```markdown
Target-branch worktree busy — cannot release. Wait for the in-flight session to finish, then re-trigger /forge-release <documentId>.
```

Leave status at `released`.

## On push rejection after merge (Step 5)

If `git push "$REMOTE" "$TARGET"` is rejected:

1. Try once: `git pull --rebase "$REMOTE" "$TARGET" && git push "$REMOTE" "$TARGET"`.
2. If rebase conflicts: treat like Step 3's conflict handler (abort, comment, transition to `reopen`).
3. If second push also fails: abort, comment "release push contested twice — escalating to human", leave status at `released`.
