6Kế hoạch triển khai

Sắp xếp theo impact / effort. Phase 1 là sweet spot <300 LOC cho 90% lợi ích.

Phase 1 — Quick wins (1–2 ngày)

  • P1.1 Restore PIPELINE_RULES + TOOL_REFERENCE vào pipeline preamble. Gọt 9 SKILL.md bỏ rules duplicate.
  • P1.2 Pre-inject issueSnapshot (title/status/plan/AC) vào user prompt — giảm 1 MCP round-trip/step.
  • P1.3 Inject sessionContext preamble khi sessionCount ≥ 1 — high-value cho forge-fix & forge-review.

Phase 2 — Medium effort (3–5 ngày)

  • P2.1 Context-save trigger ở 65% context — track token usage server-side hoặc trong Tauri runner.
  • P2.2 rollingStats cache: migration thêm cột, recompute debounced từ hooks, inject vào preamble.
  • P2.3 RAG gate (Haiku) trước forge_memory.search — flag-gated qua appConfig.useRagGate.

Phase 3 — Heavy lift (cân nhắc sau)

  • P3.1 Memory-dream consolidation cron — chỉ làm khi memory > 500 entries/project.
  • P3.2 Knowledge-graph multi-hop PageRank — cần cross-issue use case rõ.
  • P3.3 Session context embedder → graph edges → discovery cross-issue.
Chú ý Claude prompt cache Cache TTL = 5 phút, hit khi prefix system_prompt identical. Sau P1.1 preamble ~3–4K chars stable → mỗi job thứ 2+ trong cùng 5 phút tiết kiệm 90% input token. Tuyệt đối không inject issue body/sessionContext vào system prompt — đẩy vào user prompt (-p) để giữ cache hit.

Unified sessionContext schema

type SessionContext = {
  // Bookkeeping (mọi step update)
  currentState: string;
  sessionCount: number;
  lastUpdated: string;             // ISO

  // Plan/code/fix
  decisions: string[];             // cap 20
  filesModified: string[];         // cap 30
  filesIdentified: string[];       // plan-only, cap 30
  errorsResolved: string[];        // cap 20

  // Clarify
  reproEvidence?: Array<{ path: string; note: string }>;

  // Review (fix sẽ đọc)
  reviewFeedback?: Array<{
    file: string; line?: number;
    severity: 'must' | 'should' | 'nit';
    suggestion: string; addressed?: boolean;
  }>;

  // Test
  testEvidence?: Array<{ acItem: string; path: string; pass: boolean }>;

  // CI fix loop
  ciFixContext?: {
    errors: Array<{ type: string; file?: string; message: string }>;
    rawLogTail: string;            // cap 4K
  };
  ciFixAttempts?: number;          // cap 3
  ciFixHistory?: Array<{ attemptAt: string; errorSignatures: string[]; diffSignature: string }>;

  // Release
  releasedAt?: string;
  mergeCommitSha?: string;

  // Escape hatch (Tier 2 — issue-level)
  userInstructions?: string;
};
Insight chính
  • sessionContext là spinal cord — mỗi state vừa đọc vừa ghi, tạo kế thừa giữa các step.
  • forge-review BẮT BUỘC fresh CLI session (không resume code session).
  • forge-fix phụ thuộc sessionContext nặng nhất — đầu tư việc code step ghi context kỹ.
  • projectHealth chỉ cần cho plan. Inject vào state khác là phí token.
  • reproEvidence (clarify) và testEvidence (test) có thể gộp evidence[] với phase field nếu muốn dễ render UI.

Rule-based suggestion engine

Pattern matching trên aggregate. Không dùng LLM (Phase 2 mới đánh giá).

  • Block X > 40% input tokens + stddev > 30% mean → suggest cap maxChars
  • Block trong system prompt + cache miss > 30% → suggest move sang user prompt
  • State chạy Opus mà semantics đơn giản (triage / release) → suggest downgrade Sonnet/Haiku
  • Block bật mà avg tokens ≈ 0 → suggest disable (đo trong 30d window)
  • State A có cache hit thấp toàn workspace → cross-project suggestion

Apply hành vi: click Apply → PATCH appConfig.stateContext trực tiếp, ghi activity_log, không qua human approval. Rollback bằng cách click entry cũ trong activity log.

Endpoint: POST /api/analytics/apply-suggestion body { suggestionId, projectIds: string[] }. Cross-project suggestion truyền nhiều projectIds.

D. Token budgets — pre-dispatch + in-flight

Setting per-state: perRunUsd, perMonthUsd, action: warn | pause.

  • Pre-dispatch: estimate prompt cost + assume output 1.5× input. Vượt limit → halt, comment, no enqueue.
  • In-flight: track cumulative cost từ streamed usage events. Vượt → graceful_kill CLI, mark fail reason=budget_exceeded.
  • Monthly: pre-dispatch sum(cost) WHERE project=X AND state=Y AND month=current. Vượt → reject + notify.
  • 80% threshold: emit pipeline.budget_warning event → Slack / email.

Content-addressable dedup flow

// orchestrator at enqueue
const systemPrompt = await renderSystemBlocks(state, projectId, issueId);
const userPrompt   = await renderUserBlocks(state, projectId, issueId);
const hash         = sha256(systemPrompt);

await db.execute(sql`
  INSERT INTO prompt_blobs (hash, content, first_seen, ref_count)
  VALUES (${hash}, ${systemPrompt}, now(), 1)
  ON CONFLICT (hash) DO UPDATE SET ref_count = prompt_blobs.ref_count + 1
`);

await db.insert(jobs).values({
  ...,
  systemPromptHash:    hash,
  userPromptSnapshot:  userPrompt,                          // inline (unique/job)
  promptInputTokenEst: estimateTokens(systemPrompt + userPrompt),
  modelUsed:           resolveModel(state, config),
  promptBlocks:        blockBreakdown,                      // [{ id, dest, chars, tokens }]
});

Retention — .env driven

Env varDefaultEffect
FORGE_PROMPT_RETENTION_DAYS90Sau N ngày: nullify user_prompt_snapshot, archive nếu archive path set
FORGE_USAGE_RAW_RETENTION_DAYS90Raw usage_records older → aggregate monthly rollup, xóa raw
FORGE_USAGE_AGGREGATE_RETENTION_DAYS365Monthly rollup giữ N ngày
FORGE_PROMPT_ARCHIVE_PATH(unset)S3 URI hoặc local dir; unset = hard-delete khi expire

Archival flow

  1. Nightly cron archivePromptSnapshots trong core
  2. SELECT jobs WHERE archive_path IS NULL AND finished_at < now() - $retention
  3. Serialize {systemPrompt từ blob, userPrompt, blocks, usage} → JSON.gz
  4. Upload $ARCHIVE_PATH/projects/{id}/jobs/{jobId}.json.gz
  5. UPDATE jobs SET user_prompt_snapshot = NULL, archive_path = '…'
  6. Decrement prompt_blobs.ref_count; GC ref_count = 0
  7. Inspector fallback: nếu snapshot null + archive_path set → fetch on demand, cache trong session

CREATE VIEW workspace_step_cost_30d AS
SELECT project_id, state,
       sum(cost_usd) AS total_cost,
       avg(cost_usd) AS avg_cost,
       percentile_disc(0.95) WITHIN GROUP (ORDER BY cost_usd) AS p95_cost,
       sum(cached_tokens)::float / NULLIF(sum(input_tokens), 0) AS cache_hit_rate
FROM pipeline_run_step_cost
WHERE started_at >= now() - interval '30 days'
GROUP BY project_id, state;

12Locked decisions & sprint roadmap

Decisions chốt (2026-05-20)

  1. Dedup system prompt qua prompt_blobs (content-addressable sha256). Giảm ~70% storage vs inline.
  2. Retention .env-configurable, default 90 ngày. Archive sang object storage khi FORGE_PROMPT_ARCHIVE_PATH set; nếu không, hard-delete.
  3. Suggestion engine rule-based — pattern matching trên aggregate. Phase 2 mới đánh giá LLM-based.
  4. Apply suggestion không qua human approval — click → PATCH config + audit log ngay. Rollback bằng activity_log entry.
  5. Cross-project view bắt buộc — workspace dashboard so sánh projects + heatmap state × project.

Pre-Phase requirement

Phải ship Phase 1 (Section 6) trước Sprint 1. Sprint 1 phụ thuộc orchestrator gọi buildStateContext() thay vì buildJobPromptString mini. Phase 1 (P1.1 + P1.2 + P1.3) refactor sang block-based context = prerequisite cho storage snapshot.
