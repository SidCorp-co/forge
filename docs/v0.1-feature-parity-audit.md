## Notes / open questions

- **Skill custom routes** (`/skills/push-antigravity`, `/skills/push-claude`, `/skills/sync-status`, `/skills/bulk-push`) are partly subsumed by `skillSyncRoutes` + `skillRegisterRoutes` mounted at `/api/projects` in core (`index.ts:159-160`). Confirm bulk-push surface during port.
- **`/api/widget/serve`** legacy returns a JS bundle from a Strapi controller. Decision: CDN-host the prebuilt bundle (preferred) vs. re-implement a static-asset route in core. Out of scope for this audit; flagged for RFC 0003.
- **Activity eval** (`PUT /activities/:documentId/evaluate`) is the only AI-quality-feedback hook in production today. Make sure the port doesn't drop it.
- **Memory list+delete** is used by the Memory inspector page; only memory *search* is in core. Small port; bundle with the memory hardening work.
- **`/projects/health`** (project-list health summary) drives the dashboard health tiles. Confirm core's existing `/api/projects` already returns enough fields, otherwise add a thin `/api/projects/health` endpoint.
- **Tasks** are referenced by issue UI (checklist) but no longer by the pipeline runner. Verify with PM whether tasks are user-visible enough to port for v0.1 or can defer.
- **Notification mobile scope:** the mobile client only consumes `PUT /notifications/:id` (mark-as-read) â€” not list. The full surface stays in Table A; just a heads-up that the mobile dependency is narrower than web/dev.
