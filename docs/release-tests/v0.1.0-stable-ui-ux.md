## Section 19 — Notification bell + unread + mark-all-read

❌ The notification button (`<button aria-label="Notifications">`) **exists in the DOM but renders only inside `md:hidden` mobile-only top bars**. On desktop (≥768 px) every instance has `getBoundingClientRect()` width=0 height=0 because the parent containers carry `md:hidden` (i.e. `display:none` at and above the `md` breakpoint). User has no path to open the notification panel on desktop, no unread badge surfaced anywhere, and no two-tab WS broadcast can be observed. **High-impact regression.** Sidebar avatar (`A admin · Connected · System Online`) is the only WS-aware affordance on desktop.
