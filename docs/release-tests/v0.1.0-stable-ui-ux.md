**Status: BLOCKED — cannot recommend stable cut without authenticated walkthrough.**

This session completed the **unauthenticated portion** of the validation only. Section 0 pre-flight (3/4 checks) and the public-surface portion of Section 1 + Section 10 are exercised. The 19 ported feature surfaces (Tier A/B1/B2 — issue list, kanban, comments, activity, memory, dashboard, CEO, labels, schedules, skills, usage, chat-logs, notifications, agents, chat, pipeline monitor, Tauri device/index) all sit behind authentication and **could not be exercised** without the staging admin password for `admin@thejunix.com`.

Despite that, two material findings already require attention before tag:

- ❌ **Custom 404 page is gone — unknown routes 307→/login.** `/__definitely-not-a-route` redirects (HTTP 307 with `location: /login`). rc.5 §10.1 reported a custom "Error 404 — Page not found" page; that surface no longer exists. Same response shape as 401-on-protected-route, which makes the two indistinguishable to a user who hits a stale link. Regression vs rc.5.
- ❌ **Landing page logs `TypeError: Cannot read properties of undefined (reading '_gsap')` 3× on initial render.** GSAP animation library is failing to initialize on `/`. Page renders, but every animation/scrollTrigger that depends on it is dead. New regression — rc.5 listed only 1–4 generic console errors per page as advisory; this is a specific, repeatable, library-init failure.

Auth surface is otherwise healthy and one rc.5 cosmetic bug is fixed (see §1.7).

**Recommendation:** restart this walkthrough with a verified `forge_auth` cookie injected (or staging admin credentials handed to QA) and complete Sections 1.6 onward. Without that, `v0.1.0` cannot be promoted to stable.
